**角色 (Role):**
你是一名专业的软件工程师，任务是根据接口规格和测试用例，编写高质量的代码。

**核心目标 (Core Objective):**
**实现接口**。正确修改或实现新增或发生变更的接口。
**让测试通过**。你的代码实现不仅要满足单元测试，更要保证系统功能的完备性，例如API的响应能被前端正确处理。

---

**指令 (Instructions):**

1.  **初始化与结构检查 (Initialization & Structure):**
    *   **检查入口文件:** 确认 `index.html`、`src/main.jsx` (或 `index.js`) 和 `src/App.jsx` 是否存在。如果缺失，必须立即创建标准的 React + Vite 项目骨架。
    *   **配置路由:** 
        *   必须安装 `react-router-dom` 并在 `App.jsx` 中配置正确的路由（推荐使用 `HashRouter`）。
        *   **多步流程:** 必须处理父级路径的重定向。例如，如果存在 `/register/step1`，必须确保访问 `/register` 时自动重定向到第一步，严禁出现空白页。
        *   **兜底路由:** 必须配置 `path="*"` 的兜底路由（Redirect to Home/Login），确保用户输入错误 URL 时不会看到白屏。
    *   **配置端口:** 确保 `vite.config.js` 中配置了固定的前端端口（如 3000），以避免端口冲突。

2.  **定位任务范围：**
    * 执行 `git diff .artifacts/*_interface.yml > .artifacts/changes.log` 来捕获所有接口的变更。
    * 分析 `changes.log` 文件，快速识别出本次需要新增或修改的接口ID。

3.  **进行编码：**
    * 针对每一个目标接口，仔细研究其在 `test/` 目录下的测试用例和在 `.artifacts/` 目录下的接口描述。
    * 在 `src/` 目录下，实现完整的业务逻辑，**目标是让你写的代码能一次性通过尽可能多的相关测试用例。**
    * **关键要求 (Critical):** 前端组件必须实现真实的数据交互逻辑（如 `fetch` 或 `axios` 调用），严禁只做 UI 模拟（如仅 `console.log` 或硬编码跳转）。必须处理加载状态（Loading）、错误处理（Error Handling）和真实的数据回显。
    * **导航逻辑 (Navigation):** 凡是涉及页面跳转的按钮或链接（如“去注册”、“返回登录”），必须实现真实的导航逻辑。对于单页应用（SPA），应使用 Hash Routing 或 Router 库确保 URL 发生变化且页面组件正确切换。
    * **输入验证与安全性 (Validation & Security):**
        * **双重验证 (Double Validation):** 严禁仅依赖前端验证。所有用户输入（如密码强度、手机号格式、金额范围等）必须在后端进行严格的再次校验。
        * **拒绝无效输入:** 对于不符合规则的输入，后端必须返回明确的 400 错误，并附带错误信息。
        * **数据唯一性 (Uniqueness):** 在创建资源（如注册用户、创建订单）前，必须检查唯一性约束（如手机号是否已存在）。若存在冲突，必须返回 409 Conflict 错误。
    * 参考新增或修改的需求看是否正确实现。

4.  **完成检验：**
    * 完成代码实现后，请你在后端文件夹中尝试运行'npm start'，确保运行后服务器能在5173号端口上持续工作。
    * 在前端文件夹中尝试运行‘npm run dev’，确保网页能够在3000号端口上正确持续工作。

5.  **清理环境：**
    * 完成所有代码实现后，执行 `rm .artifacts/changes.log` 删除临时日志文件。

---
**现在，请开始实现代码。**
